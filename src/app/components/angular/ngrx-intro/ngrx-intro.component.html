NgRx là gì?


NgRx là một framework để xây dựng các ứng dụng reactive trong Angular. Cung cấp một số tính năng:

  • Quản lý global và local state.
  • Cô lập các side effects để tạo ra kiến trúc phần mềm clean hơn.(Cô lập các hiệu ứng phụ - là việc tách các hiệu ứng phụ—những hoạt động tương tác với các hệ thống bên ngoài như API, lưu trữ cục bộ hoặc các dịch vụ xử lý tác vụ bất đồng bộ—ra khỏi logic cốt lõi của các thành phần Angular. Việc này rất quan trọng để duy trì một kiến trúc thành phần sạch hơn.)
  • Quản lý Entity collection
  • Tích hợp với Angular Router
  • Công cụ cho developer giúp nâng cao trải nghiệm dev khi build nhiều loại ứng dụng khác nhau.

Packages

NgRx packages được chia thành một số loại chính.

State
  • Store - Global state management cho Angular apps được hỗ trợ bởi RxJS, lấy cảm hứng từ Redux
  • Effects - Side effect model cho @ngrx@store
  • Router Store - các ràng buộc để kết nối Angular Router với @ngrx@store
  • Entity - Entity State adapter để quản lý record collections
  • ComponentStore - Thư viện độc lập để quản lý local/component state.
  • Signals - Reactive store và bộ tiện ích cho Angular Signals.
  • Operator - Toán tử RxJS được chia sẻ cho thư viện NgRx.
Data

Data - Tiện ích mở rộng để đơn giản hóa việc quản lý entity data.

View

Component - Tiện ích mở rộng để xây dựng reactive Angular templates.

Developer Tools

  • Store Devtools - Công ccuj dành cho @ngrx/store cho phép theo dõi trực quan state và debugging.

  • Schematics - Scaffolding library(là quá trình tự động tạo ra các cấu trúc cơ bản của một thư viện, giúp cho việc phát triển và tổ chức mã nguồn trở nên dễ dàng hơn.) cho các ứng dụng Angular sử dụng NgRx.

  • ESLint Plugin - Quy tắc ESLint để cảnh báo các hành vi xấu. Nó cũng chứa một số bản tự động sửa lỗi để code nhất quán.
