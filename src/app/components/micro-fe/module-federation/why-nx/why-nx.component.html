Lợi ích của Nx

Có thể sử dụng Nx để:

  • Tăng tốc build, test trên cả local và CI(cả standalone và monorepo application)
  • Nhanh chóng xây dựng ra 1 dự án mới(sử dụng các plugin của Nx).
  • Dễ dangg tích hợp công cụ mới(Storybook, Tailwind, ...) vào project hiện tại.
  • Đảm bảo tính nhất quán và chất code bằng custom generators and lint rules
  • Hỗ trợ cập nhật frameworks and tools bằng tính năng migration
Nx hoạt động như thế nào?

Nx được xây dựng theo kiểu modular cho phép bạn chỉ sử dụng những tính năng bạn cần.

Nx package cung cấp tính năng cơ bản như: workspace analysis, task running, caching, distribution, code generation và automated code migrations.

Plugins là các NPM package được xây dựng trên Nx package. Nx plugins chứa code generators, executors(để trừu tượng hóa các lower-level build tooling) và tự động code migrations. Trái ngược với gói Nx, hoạt động theo cách tương tự với bất kỳ dự án JS hoặc không phải JS nào, các plugin thường dành riêng cho công nghệ. Ví dụ: @nx/react bổ sung hỗ trợ xây dựng ứng dụng React và lib, @nx/cypress bổ sung khả năng kiểm tra e2e với Cypress. Các plugin giúp nhà phát triển làm việc hiệu quả hơn bằng cách loại bỏ mọi trở ngại khi tích hợp các công cụ khác nhau với nhau và bằng cách cung cấp các tiện ích để cập nhật chúng. Nhóm Nx duy trì các plugin cho React, Next, Remix, Angular, Jest, Cypress, Storybook, v.v. Bạn có thể sử dụng gói @nx/plugin để dễ dàng tạo một plugin mới hoặc thậm chí chỉ tự động hóa không gian làm việc cục bộ của bạn. Ngoài ra còn có hơn 80 plugin cộng đồng.
